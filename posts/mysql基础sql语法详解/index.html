MySQL基础SQL语法详解

所有实例基于统一测试表，便于边学边练、加深记忆。

一、前置准备：测试表与测试数据

后续所有SQL语句均基于以下两张测试表（学生表、班级表），先执行以下语句创建表并插入测试数据，确保所有实例可直接复制运行。

-- 1. 创建学生表（核心演示表，存储学生基本信息）
CREATE TABLE student (
    id INT PRIMARY KEY AUTO_INCREMENT,  -- 学生ID，主键（唯一标识），自增（自动生成）

二、SQL语句分类（必记）

SQL（Structured Query Language，结构化查询语言）是操作关系型数据库的标准语言，按功能可分为4大类，基础课程重点掌握前3类，尤其是数据查询语言（DQL）。

  • DQL（数据查询语言）：核心语句为SELECT，用于查询数据库中的数据，是最常用、最重要的SQL语句，也是本文重点讲解内容。

  • DML（数据操作语言）：核心语句为INSERT（新增）、UPDATE（修改）、DELETE（删除），用于操作表中的数据。

  • DDL（数据定义语言）：核心语句为CREATE（创建）、ALTER（修改）、DROP（删除），用于操作表的结构（如建表、改字段、删表）。

  • DCL（数据控制语言）：核心语句为GRANT（授权）、REVOKE（回收权限），用于管理数据库权限，基础课程了解即可。

三、核心语法详解（按常用优先级排序）

（一）DQL 数据查询语言（重点中的重点）

数据查询是SQL的核心，掌握以下内容，可应对日常80%以上的查询需求，从简单到复杂逐步讲解，搭配实例帮助理解。

1. 基础查询（无筛选、无排序）

用于查询表中指定字段或所有字段的数据，语法简单，是所有查询的基础。

-- 1. 查询表中所有字段（* 代表所有字段，效率较低，适合快速查看数据）
SELECT * FROM student;

-- 2. 查询指定字段（推荐用法，效率高，明确需要的信息）
SELECT name, age, score FROM student;

-- 3. 给字段起别名（AS可省略，别名用于简化显示，更易读）
SELECT name AS 姓名, age 年龄, score 成绩 FROM student;

2. DISTINCT 去重查询

作用：去除查询结果中完全重复的行，只保留唯一值，常用于统计不重复的数据（如不重复的班级、不重复的性别）。

-- 1. 查询所有班级（未去重，会出现重复的班级名称）
SELECT class FROM student;

-- 2. 查询不重复的班级（去重，只保留每个班级一次）
SELECT DISTINCT class FROM student;

-- 3. 多字段联合去重（只有两个字段完全相同，才会被去重）
SELECT DISTINCT class, gender FROM student; -- 统计每个班级的不同性别

重点：DISTINCT必须紧跟在SELECT后面，仅对查询的字段生效，只会去除“完全相同”的行。

3. 条件查询（WHERE子句）

用于筛选符合条件的数据，配合运算符使用，是查询中最常用的筛选方式。先掌握常用运算符，再看实例。

常用运算符

运算符类型 具体运算符 含义
比较运算符 = 等于
> / < 大于 / 小于
>= / <= 大于等于 / 小于等于
!= / <> 不等于（两种写法均可）
BETWEEN ... AND ... 区间查询（包含首尾值）
IS NULL / IS NOT NULL 判断字段是否为空（NULL不能用=判断）
逻辑运算符 AND 同时满足多个条件
OR 满足任意一个条件即可
特殊运算符 IN 多值匹配，替代多个OR
LIKE 模糊查询，配合通配符使用

条件查询实例（重点掌握）

-- 1. 基础比较查询：查询成绩≥80分的学生
SELECT * FROM student WHERE score >= 80;

-- 2. 逻辑运算符：查询高一1班的男生
SELECT * FROM student WHERE class = '高一1班' AND gender = '男';

-- 3. IN多值匹配：查询班级为高一1班、高一2班的学生（替代OR，更简洁）
SELECT * FROM student WHERE class IN ('高一1班', '高一2班');
-- 等价于：SELECT * FROM student WHERE class = '高一1班' OR class = '高一2班';

-- 4. BETWEEN区间查询：查询年龄在17~18岁之间的学生（包含17和18）
SELECT * FROM student WHERE age BETWEEN 17 AND 18;

-- 5. LIKE模糊查询（配合通配符）
-- 通配符1：% 匹配任意长度的字符（0个、1个、多个均可）
SELECT * FROM student WHERE name LIKE '张%'; -- 查询姓“张”的学生（张开头）
SELECT * FROM student WHERE name LIKE '%五%'; -- 查询名字中包含“五”的学生
SELECT * FROM student WHERE name LIKE '%八'; -- 查询名字以“八”结尾的学生

-- 通配符2：_ 匹配单个任意字符（必须是1个）
SELECT * FROM student WHERE name LIKE '张_'; -- 查询名字为两个字、且姓张的学生

-- 6. 空值判断：查询成绩不为空的学生（NULL不能用=判断）
SELECT * FROM student WHERE score IS NOT NULL;

4. 排序查询（ORDER BY子句）

用于对查询结果按指定字段排序，可升序或降序，常用于排行榜、数据排序展示。

-- 1. 按成绩升序排序（ASC：升序，默认可省略）
SELECT * FROM student ORDER BY score ASC;

-- 2. 按成绩降序排序（DESC：降序，必须写）
SELECT * FROM student ORDER BY score DESC;

-- 3. 多字段排序：先按班级升序，班级相同则按成绩降序
SELECT * FROM student ORDER BY class ASC, score DESC;

5. 分页查询（LIMIT子句）

用于限制查询结果的条数，避免一次性查询全表数据（尤其是数据量较大时），是日常开发和查询中非常常用的语句。

-- 语法：LIMIT 起始索引, 查询条数（起始索引从0开始，可省略，省略时从0开始）
-- 实例1：查询前3条学生数据（起始索引0，查询3条）
SELECT * FROM student LIMIT 0, 3;

-- 实例2：简化写法（省略起始索引，默认从0开始）
SELECT * FROM student LIMIT 3; -- 等价于上面的语句

-- 实例3：查询第4~6条数据（起始索引3，查询3条）
SELECT * FROM student LIMIT 3, 3;

6. 聚合函数（统计数据）

用于对数据进行统计计算，常用的5个聚合函数，必须配合SELECT使用，可单独使用，也可配合分组查询。

聚合函数 含义
COUNT(*) 统计总行数（最常用，包含非空字段）
SUM(字段名) 计算指定字段的总和（仅适用于数值类型）
AVG(字段名) 计算指定字段的平均值（仅适用于数值类型）
MAX(字段名) 查询指定字段的最大值（仅适用于数值类型）
MIN(字段名) 查询指定字段的最小值（仅适用于数值类型） 
-- 实例1：查询学生总人数
SELECT COUNT(*) AS 总人数 FROM student;

-- 实例2：查询全班总成绩、平均成绩、最高分、最低分
SELECT 
    SUM(score) AS 总成绩,
    AVG(score) AS 平均成绩,
    MAX(score) AS 最高分,
    MIN(score) AS 最低分
FROM student;

-- 实例3：查询高一1班的平均成绩
SELECT AVG(score) AS 高一1班平均分 FROM student WHERE class = '高一1班';

7. 分组查询（GROUP BY子句）

用于按指定字段分组，对每组数据进行统计（必须配合聚合函数使用），常用于按类别统计数据（如按班级统计人数、按性别统计平均分）。

-- 实例1：按班级分组，统计每个班级的人数、平均成绩
SELECT 
    class AS 班级,
    COUNT(*) AS 人数,
    AVG(score) AS 平均成绩
FROM student
GROUP BY class;

-- 实例2：按班级和性别分组，统计每组的人数
SELECT 
    class AS 班级,
    gender AS 性别,
    COUNT(*) AS 人数
FROM student
GROUP BY class, gender;

8. 分组后筛选（HAVING子句）

用于筛选分组后的统计结果，注意区分WHERE和HAVING的区别：WHERE筛选的是原始数据，HAVING筛选的是分组后的统计结果。

-- 实例：查询平均成绩≥80分的班级（先分组，再筛选分组结果）
SELECT 
    class AS 班级,
    AVG(score) AS 平均成绩
FROM student
GROUP BY class
HAVING AVG(score) >= 80;

-- 错误写法：用WHERE筛选分组结果（WHERE不能直接筛选聚合函数结果）
-- SELECT class, AVG(score) FROM student WHERE AVG(score) >=80 GROUP BY class;

9. LEFT JOIN 左外连接（多表查询基础）

当需要查询多个表的数据时，使用连接查询，LEFT JOIN是最常用的连接方式，核心规则：以左表为基准，左表所有数据全部显示，右表只显示与左表匹配的数据，匹配不到的地方显示NULL。

-- 语法：SELECT 字段 FROM 左表 别名 LEFT JOIN 右表 别名 ON 左表.关联字段 = 右表.关联字段;
-- 实例：查询所有学生的信息，以及他们对应的班级信息（左表：student，右表：class）
SELECT 
    s.id AS 学生ID,
    s.name AS 姓名,
    s.class AS 班级名称,
    c.cid AS 班级ID
FROM student s -- s是student表的别名，简化写法
LEFT JOIN class c -- c是class表的别名
ON s.class = c.cname; -- 关联条件：学生表的班级名称 = 班级表的班级名称

-- 结果特点：所有学生都会显示（左表全保留），高一3班的周九会显示班级ID，未匹配到的班级会显示NULL

10. DQL查询执行顺序

写复杂查询时，必须记住执行顺序，否则容易出现语法错误或查询结果不符合预期，顺序如下（从左到右）：

  1. FROM：确定查询的表（如果有连接，先执行连接）

  2. LEFT JOIN：执行左连接，匹配两张表的数据

  3. WHERE：筛选原始数据（分组前筛选）

  4. GROUP BY：对筛选后的数据进行分组

  5. HAVING：筛选分组后的统计结果（分组后筛选）

  6. SELECT：查询指定字段（包含聚合函数、DISTINCT去重）

  7. ORDER BY：对查询结果进行排序

  8. LIMIT：限制查询结果的条数

11. 综合查询实例

结合以上所有知识点，完成一个综合查询需求，帮助巩固记忆，理解各语句的配合使用。

-- 需求：
-- 1. 查询高一1班、高一2班的学生
-- 2. 姓名包含“四”或“六”，成绩大于70分
-- 3. 去除重复的班级和性别组合
-- 4. 关联班级表，显示学生姓名、班级、性别、成绩、班级ID
-- 5. 按成绩降序排序，只取前4条数据
SELECT DISTINCT 
    s.name AS 姓名,
    s.class AS 班级,
    s.gender AS 性别,
    s.score AS 成绩,
    c.cid AS 班级ID
FROM student s
LEFT JOIN class c ON s.class = c.cname
WHERE 
    s.class IN ('高一1班', '高一2班')
    AND (s.name LIKE '%四%' OR s.name LIKE '%六%')
    AND s.score > 70
ORDER BY s.score DESC
LIMIT 4;

（二）DML 数据操作语言（增删改）

用于操作表中的数据，重点注意：修改（UPDATE）和删除（DELETE）必须加WHERE条件，否则会操作全表数据，造成数据丢失！

1. 插入数据（INSERT INTO）

用于向表中新增一条或多条数据，语法灵活，可指定字段，也可省略字段（需与表结构顺序一致）。

-- 1. 插入单条数据（指定字段，推荐，灵活，即使表结构变化也不易出错）
INSERT INTO student(name, age, gender, class, score) 
VALUES ('吴十', 17, '女', '高一2班', 90.0);

-- 2. 插入多条数据（高效，一次插入多条，减少操作次数）
INSERT INTO student(name, age, gender, class, score) 
VALUES 
('郑十一', 18, '男', '高一1班', 82.5),
('王十二', 17, '女', '高一3班', 76.0);

-- 3. 省略字段插入（需与表结构字段顺序一致，主键自增可省略）
INSERT INTO student VALUES (NULL, '冯十三', 19, '男', '高一3班', 89.0);

2. 修改数据（UPDATE）

用于修改表中已有的数据，必加WHERE条件，否则会修改表中所有数据。

-- 实例1：修改张三的成绩为90.0（指定单个字段修改）
UPDATE student SET score = 90.0 WHERE name = '张三';

-- 实例2：修改王五的年龄和班级（多字段修改，用逗号分隔）
UPDATE student SET age = 19, class = '高一3班' WHERE id = 3;

-- 错误写法（无WHERE条件，会修改全表学生的成绩）
-- UPDATE student SET score = 100;

3. 删除数据（DELETE）

用于删除表中的数据，必加WHERE条件，否则会清空整个表的数据（不可恢复，谨慎操作）。

-- 实例1：删除成绩<60分的学生（筛选条件删除）
DELETE FROM student WHERE score < 60;

-- 实例2：删除指定ID的学生（最安全的删除方式，ID唯一）
DELETE FROM student WHERE id = 10;

-- 错误写法（无WHERE条件，会清空整个student表）
-- DELETE FROM student;

（三）DDL 数据定义语言（表结构操作）

用于操作表的结构，基础课程掌握常用的建表、改表、删表操作即可，操作前需谨慎（会影响表结构和数据）。

1. 创建表（CREATE TABLE）

语法：CREATE TABLE 表名 (字段名 数据类型 约束条件, ...);

-- 示例：创建一个教师表（teacher）
CREATE TABLE teacher (
    tid INT PRIMARY KEY AUTO_INCREMENT, -- 教师ID，主键，自增
    tname VARCHAR(20) NOT NULL,         -- 教师姓名，非空
    subject VARCHAR(20),                -- 所教科目
    age INT                             -- 教师年龄
);

2. 修改表结构（ALTER TABLE）

用于新增字段、修改字段、删除字段，常用操作如下：

-- 1. 给student表新增“手机号”字段
ALTER TABLE student ADD phone VARCHAR(11);

-- 2. 修改student表中age字段的数据类型（从INT改为TINYINT，更节省空间）
ALTER TABLE student MODIFY age TINYINT;

-- 3. 删除student表中的“手机号”字段
ALTER TABLE student DROP phone;

3. 删除表（DROP TABLE）

用于删除整个表（包括表结构和表中所有数据），谨慎操作，删除后不可恢复。

-- 语法：DROP TABLE IF EXISTS 表名;（IF EXISTS避免表不存在时报错）
DROP TABLE IF EXISTS teacher; -- 删除教师表

（四）DCL 数据控制语言（了解）

用于管理数据库权限，基础课程了解即可，无需深入掌握。

-- 示例1：给用户授予student表的查询权限
GRANT SELECT ON student TO 'user1'@'localhost';

-- 示例2：回收用户的查询权限
REVOKE SELECT ON student FROM 'user1'@'localhost';

四、重点总结与注意事项

1. 核心优先级

DQL（查询）&gt; DML（增删改）&gt; DDL（表结构）&gt; DCL（权限），重点掌握DQL查询，这是日常使用最多的内容。

2. 安全红线

  • UPDATE和DELETE语句，必须加WHERE条件，否则会操作全表数据，造成不可恢复的损失。

  • DROP TABLE语句需谨慎，删除前确认是否需要保留数据。

  • NULL值判断必须用IS NULL / IS NOT NULL，不能用=或!=。

3. 高频组合用法

日常查询中，最常用的组合：SELECT + WHERE + ORDER BY + LIMIT（筛选+排序+分页），熟练掌握这一组合，可应对大部分查询需求。

4. 学习建议

所有实例均可直接复制到MySQL中运行，边练边记效果最好；重点理解DQL查询的逻辑和执行顺序，多做综合练习，巩固知识点。
